# 言語の基本

> 🌐 [English](../02-language-basics.md) · **日本語**

この章では、式指向のコア（束縛、型、関数、制御フロー）について一気に解説します。この章を読み終えれば、どんな Align の関数でも読めるようになるはずです。

## 束縛

```align
fn main() -> i32 {
    x := 10             // introduce a binding (immutable)
    y: i64 := 20        // with an explicit type annotation
    mut total := 0      // mutable — only `mut` bindings may be reassigned
    total = x + y       // reassign with `=`, not `:=`
    print(total)        // 30
    return 0
}
```

`:=` は変数の導入、`=` は再代入です。`mut` でない変数への再代入はコンパイルエラーになります。つまり、変数が可変かどうかは常に宣言の時点で明確になります。型は推論されますが、推論結果とは異なる型にしたい場合や、コードの意図を明確にしたい場合には型注釈を付けてください。

## 文と行

Align は Go 言語のスタイルを採用しています。改行によって文が終了し、`;` （セミコロン）は複数の文を1行にまとめるためにのみ使用されます。ブロックは波かっこ `{}` で区切られるため、インデント自体に意味はありません。`.` または二項演算子で始まる行は、前の行からの続きとして扱われます。この仕組みによって、長いパイプライン処理を複数行に折り返すことができます。

```align
fn main() -> i32 {
    total := [1, 2, 3]
        .map(fn x { x * 2 })
        .sum()
    print(total)        // 12
    return 0
}
```

## 数値型

符号付きの `i8 i16 i32 i64`、符号なしの `u8 u16 u32 u64`、浮動小数点の `f32 f64`、それに `bool` と `char` (Unicode スカラー値、`'A'`、`'あ'`) があります。型が指定されていない整数リテラルはデフォルトで `i64`、浮動小数点リテラルはデフォルトで `f64` になります。**暗黙の型変換は一切行われません**。サイズの異なる型を混ぜて演算しようとすると型エラーになるため、変換は `as` を使って明示する必要があります。

```align
fn main() -> i32 {
    x: i8 := 127
    y := x + 1          // i8 arithmetic
    print(y)            // -128 — overflow wraps, defined two's-complement
    big := 300
    b := big as i8      // explicit narrowing
    print(b)            // 44 (300 mod 256) — and the compiler warns: lossy conversion
    return 0
}
```

この例には、意図的な決定が 2 つ込められています。

- **整数のオーバーフローはラップ（折り返し）する**: これは2の補数表現における定義された動作であり、未定義動作や隠れたバグの原因ではありません。オーバーフローを検知したり、飽和させたりする演算が必要な場合は、明示的な `checked_*`、`saturating_*`、`wrapping_*` メソッドが用意されており、プログラマの意図がソースコード上に明確に現れるようになっています。
- **縮小変換 (narrowing) は明示され、監査される**: `as` による切り捨ては定義された動作です。コンパイラは情報の損失が発生する `as` 変換すべてに対して警告を出すため、意図しないデータの切り捨てが紛れ込むのを防ぎます。

ゼロ除算（および `%` 演算でのゼロによる剰余）は、実行時のハードエラーとなります。プログラムは直ちに終了（アボート）し、気付かないうちに誤った計算結果を返すようなことは決してありません。また、範囲外のリテラル（例：`x: i8 := 200`）はコンパイルエラーになります。

## すべては式

`if`、`match`、ブロック —— これらはすべて値を生みます。ブロックの値は、その末尾の式です。

```align
fn main() -> i32 {
    limit := 100
    fee := if limit > 50 { 10 } else { 25 }   // if is an expression
    x := {
        a := 3
        a * 2                                  // trailing expression = block's value
    }
    print(fee + x)                             // 16
    return 0
}
```

このため Align には三項演算子が存在せず、`if` に文と式の区別もありません。`if` は1種類のみであり、その結果を変数に代入したり、関数の戻り値として使うことができます。

## 関数

```align
fn add(a: i64, b: i64) -> i64 {
    return a + b
}

// single-expression form: `=` instead of a block
fn square(x: i64) -> i64 = x * x

fn main() -> i32 {
    print(add(square(3), 1))    // 10
    return 0
}
```

関数の本体の書き方は2種類のみです。`return` を含むブロック `{}` を使うか、単一の式を返すための `= expr` を使うかです。引数はデフォルトで不変（イミュータブル）です。小さな値はコピーされて渡されます。所有権を持つ型がどのように扱われるかについては [05](05-memory.md) 章で解説します。

## `for` はない

Align には `for` も `while` もありません。これは機能の欠落ではなく、言語の設計思想の核となる部分です。データに対する反復処理は**パイプライン** (`xs.map(f).where(p).sum()`、詳しくは [06](06-pipelines.md) 章を参照) として記述し、コンパイラはそれをベクトル化可能な単一のループへと融合します。コレクションを手作業で走査するコードを書きそうになったら、一度立ち止まってください。代わりに、どのような変換を行いたいのかを考え、それに名前を与えてください。

それでも残る反復処理、つまり「反復回数が反復処理自体によって決まるもの」（例えば、EOF まで読み込む、成功するまでリトライするなど）に対しては、専用の構文が1つだけ用意されています。それが **`loop` 式**です。これは `break` が実行されるまでブロックを繰り返し、`break value` によってその値を返してループを終了します。したがって、`loop` も `if` や `match` と同様に式として扱われます。

```align
mut total := 0
n_read := loop {
    n := r.read(buf)?               // errors still exit the function, as always
    if n == 0 { break total }       // break is the only loop exit
    total = total + n
}
```

`continue` やラベル付きの break はありません。次の反復へスキップしたい場合は、ループ本体の残りの部分を `if` で囲んでください。また、多重ループから一気に脱出したいような場合は、その処理自体を別の関数として切り出すべきサインです。ループの状態管理は、ループの前に宣言した `mut` ローカル変数で行います。

役割分担は明確です。**パイプラインはデータの流れを扱い、`loop` は制御フローを扱います。** ループを書きたい衝動に駆られたら、まず「どのようなデータ変換を行おうとしているのか」を考えてください。多くの場合、それはパイプラインで表現できます。なお、`loop` の中で配列をインデックスで走査しようとすると lint 警告が出ます。（再帰呼び出しも可能ですが、それはパーサや木構造の探索など、真に再帰的な問題のためのものであり、単純な反復処理には使用すべきではありません。Align は末尾呼び出しの最適化を保証していないため、再帰を使った「ループ」はスタックを消費し続けます。）

## 名前付き定数

```align
WIDTH: i32 := 6
HEIGHT: i32 := 7
AREA: i32 := WIDTH * HEIGHT     // folded at compile time

fn main() -> i32 = AREA         // exits 42
```

トップレベルで `NAME := expr` と記述すると、コンパイル時定数になります。Align の他の機能と同様に専用のキーワードは不要です。定数は不変であり、使用される各箇所でコンパイル時に値が展開（インライン化）されます。定義の順序は問いません（後で定義される定数を参照することも可能です）。型注釈のない整数定数は `i64` になります。

定数は**配列リテラル**にもできます。コンパイル時のルックアップテーブルになります:

```align
PRIMES := [2, 3, 5, 7, 11]      // slice<i64>
DAYS := ["Mon", "Tue", "Wed"]   // slice<str>

fn main() -> i32 = PRIMES[3] as i32   // 7 で終了する
```

定数配列の型は **`array<T>` ではなく `slice<T>`** になります。Align では、所有権は型の性質として扱われますが、トップレベルの定数はデータへの所有権を持ちません。文字列リテラル `"hello"` が、所有権を持つ `string` ではなく、読み取り専用のバイト列へのビューである `str` となるのと同様に、`PRIMES` は実行ファイルに組み込まれた読み取り専用テーブルへのビュー (`slice<i64>`) となります。

インデックスアクセス (`PRIMES[3]`)、要素数の取得 (`.len()`)、パイプライン処理 (`PRIMES.where(...).sum()`) はすべて、メモリの動的確保なしにこのテーブルを直接読み取ります。また、定数インデックスでのアクセスはコンパイル時に評価されます。テーブルは読み取り専用であるため、内容を書き換えることはできません（`PRIMES[0] = 9` はコンパイルエラーになります）。内容を変更したい場合は、まず所有権を持つ配列 (`array<T>`) にコピーする必要があります。型注釈は `slice<T>` と記述するか、省略してください。`array<T>` という注釈はエラーになります。

## `print` とテンプレート文字列

`print(x)` はプリミティブな値と改行を書き出します。整数、浮動小数点数(`1.0` は `1.0` と、最短の往復表現で表示されます)、`bool`(`true`/`false`)、`char`、そして文字列です。テキストを組み立てるには**テンプレート文字列**があります。

```align
fn main() -> i32 {
    name := "align"
    score := 40
    print(template "Hello {name}, score={score + 2}")   // Hello align, score=42
    return 0
}
```

テンプレート文字列内の波かっこ `{}` の中には、任意の式を書くことができます。文字列に関する詳細（なぜ `builder` が存在するのか、文字列の結合 `+` はいつ許可されるのかなど）については、[07](07-strings-and-text.md) 章で解説します。
