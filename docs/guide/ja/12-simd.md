# 明示的な SIMD ― vecN、マスク、アライメント

> 🌐 [English](../12-simd.md) · **日本語**

パフォーマンス最適化に関する章の中でも、この章は最後に読んでください。最大の教訓が「**たいていの場合、この機能は必要ない**」だからです。スライスや SoA の列に対するパイプライン処理はコンパイラによって自動でベクトル化されます。それが Align における本道であり、ハードウェアの SIMD 幅に依存せず、プログラムが実行されるプロセッサの性能を自動的に引き出します。`vecN<T>` は、自動ベクトル化ではカバーしきれない残りの1%のケース、すなわち「特定の固定幅でレーンごとの処理を厳密に書き下したいカーネル」のための抜け道です。

## `vecN<T>` ― 値としての SIMD レジスタ

```align
fn main() -> i32 {
    a: vec4<i32> := [1, 2, 3, 4]
    b: vec4<i32> := [10, 20, 30, 40]
    c := a + b          // one instruction: [11, 22, 33, 44]
    d := c * 2          // scalar broadcasts across lanes
    print(d[2])         // 66 — lane read (constant index)
    print(c.max())      // 44 — horizontal reduction
    return 0
}
```

数値スカラー型に対して、`vec2`、`vec4`、`vec8`、`vec16` の SIMD ベクトル型が提供されています。算術演算はレーン（要素）単位で行われ、1つの演算が1つの SIMD 命令にコンパイルされます。スカラー値と演算すると全レーンにブロードキャストされます。`v[i]` で特定のレーンの値を読み取ることができ、`mut` 宣言されたベクトルなら `v[1] = 99` のようにレーンへの直接書き込みも可能です。

`a.min(b)` や `a.max(b)` のような要素単位の比較のほか、浮動小数点演算（`sqrt`、`abs`、`floor`、`ceil`、`round`、`trunc`）もレーンごとに適用されます。また、古典的なカーネル処理をカバーするために、2つのフリー関数が提供されています。内積を計算する `dot(a, b)` と、FMA（Fused Multiply-Add：丸め誤差が1回で済む融合積和演算）を行う `fma(a, b, c)` です。

整数ベクトルの演算セマンティクスは、スカラー演算の挙動と完全に一致します。オーバーフローが発生した場合はラップアラウンド（切り捨て）し、ゼロ除算が発生した場合はプログラムがアボート（強制終了）します。どのレーンの演算であっても未定義動作（Undefined Behavior）にはなりません。

## マスクと `select` ― 構造からして分岐なし

ベクトル同士の比較結果は `maskN<T>`（マスク型）になります。これは `if` のような制御フロー（分岐）としてではなく、データ（ブレンド用のフラグ）として使用します。

```align
fn main() -> i32 {
    scores: vec4<i32> := [90, 45, 82, 60]
    m := scores > 80                        // mask4<i32>: [t, f, t, f]
    picked := select(m, scores, scores * 0) // lane-wise blend, no branch
    print(picked.sum_where(m))              // 172 — masked reduction: 90 + 82
    return 0
}
```

`select(mask, a, b)` はマスクの真偽に応じてレーンの値をブレンドします。`v.sum_where(mask)` は、マスクが `true` ととなっているレーンのみを対象に値をリダクション（集計）します。これは、パイプラインの `where(p)` が内部でどのようにコンパイルされているかを、明示的にコードとして書き下したものです。`xs.where(p).sum()` が分岐なしで実行されるのと同じ「アイデンティティ・セレクト（identity-select）」の技法です。

## メモリとレジスタを橋渡しする ― `load` / `store`

```align
fn scale_add(xs: slice<f64>, ys: slice<f64>, out dst: slice<f64>) {
    x: vec4<f64> := xs.load(0)      // 4 lanes from the slice, bounds-checked
    y: vec4<f64> := ys.load(0)
    dst.store(0, x * 2.0 + y)       // two instructions, 4 lanes each
}

fn main() -> i32 {
    xs := [1.0, 2.0, 3.0, 4.0]
    ys := [10.0, 10.0, 10.0, 10.0]
    mut out := [0.0, 0.0, 0.0, 0.0]
    mut d: slice<f64> := out
    scale_add(xs, ys, d)
    print(out.sum())                // 60.0
    return 0
}
```

`s.load(i)` は、スライスから N 個の連続する要素を SIMD レジスタに読み込みます（N はベクトルの型注釈から推論されます）。`s.store(i, v)` は、計算結果のベクトルを `out` スライス越しにメモリへ書き戻します。これらの操作はどちらも実行時に境界チェック（Bounds Check）が行われます。この `load`/`store` のペアこそが、手書きのカーネルが配列を走査する際の基本形です。通常は `chunks(N)` で処理を進め、最後に N に満たない端数要素をスカラー処理で片付ける、というパターンになります。

## `align(N)` ― ロードをアライメントさせたいとき

`align(64) xs := [...]` は配列のメモリ領域を 64 バイト境界に過剰にアライメント（Over-alignment）し、`align(64) CacheLine { ... }` は構造体自体をアライメントします（配列のストライドもこれに合わせてパディングされます）。コンパイラが「アライメントされたオフセットからのロードである」と証明できた場合、ベクトルロード命令はより広範なアライメントを前提とした最適化を行うようになります。また、FFI（Foreign Function Interface）で C 言語などと共有するバッファのために、`layout(C)` と組み合わせて使用することも可能です。ただし、これは現実のハードウェアで効果があるとはいえ、見返りの小さなマイクロ最適化です。適用する前に必ずベンチマークを計測してください。

## 二層のルール

- **第1層（デフォルト）：パイプライン。** `xs.map(f).where(p).sum()` のような処理、SoA 列への操作、`group_by` などを指します。これは SIMD の幅に依存しません。コンパイラがターゲット・アーキテクチャに合わせて最適なレーン数を選択し、同じソースコードから AVX2、NEON、あるいはスケーラブルなベクタ ISA 用に自動ベクトル化を行います。この層は、あなたのデータサイズやハードウェアに自動で追随してくれます。
- **第2層（抜け道）：`vecN` / `maskN`。** ドット積、FIR フィルタ、ブレンド処理などのカーネルを記述する際に、プログラマ自身が選択した「固定幅のレジスタの動き（振り付け）」を厳密に制御したい場合に使用します。

> **Optimization:** 第1層のvectorizeはtargetとprofileに依存し、言語上の保証でも固定byte閾値でもありません。loopの合法性と入力の形にも左右されます。判断は `alignc explain-opt`、得られたvector幅は `emit-llvm --stage optimized` で確認できます。

もし第2層（明示的な SIMD）に手を伸ばすのであれば、上記の `scale_add` のように「スライスを境界とする1つの小さな関数」の中にロジックを閉じ込めてください。呼び出し側が「この関数が内部で手動ベクトル化されていること」を意識する必要がないようにすべきです。そして、第1層の自動ベクトル化をあきらめる前に、必ずコードを監査（プロファイル）してください。`alignc emit-llvm` コマンドを使えば、パイプラインがすでに生成している LLVM IR の中に `<4 x i64>` のようなベクトル型が見つかるはずです。この監査を行った結果、最もよくある結末は「手書きした SIMD カーネルを削除して、元のパイプラインに戻す」ことです。
