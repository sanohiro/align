# パイプラインで考える

> 🌐 [English](../02-pipelines.md) · **日本語**

これが Align の核心です。`for` ループを書くことはめったになく、代わりにコレクションに対する変換を記述し、コンパイラにループを生成させます — 融合され(fused)、分岐が最小化され、ベクトル化可能な形で。

## 基本の形

```align
total := prices.map(with_tax).where(in_stock).sum()
```

左から右に読んでください。`prices` を受け取り、各要素に `with_tax` を適用し、`in_stock` が成り立つものだけを残し、結果を合計する。ここで重要なのは、**中間の配列が一切作られない**ことです — `map`、`where`、`sum` は単一の counted loop(回数の決まったループ)へと融合されます。これはストリーム融合(stream fusion)であり、あなたではなくコンパイラが行います。

## ステージ(段階)

- `map(f)` — 各要素を変換する。
- `where(p)` — 述語(predicate)に一致する要素だけを残す。(`where(.active)` は `active` フィールドが true である行だけを残す。)
- `.field` — 各構造体からフィールドを射影(project)する。
- `reduce(init, f)` / `sum()` — 単一の値へと畳み込む(collapse)。パイプラインは reduction(還元)で終わる。
- `chunks`、`partition` — より広い一群(`sort` や `scan` のようなものが、言語の成長とともにさらに加わっていく。現在の一覧は仕様書を参照)。

## なぜこれが速いのか

スカラー言語で手書きしたバージョンは、各ステップごとに一時配列を allocate し、メモリを3回走査します。Align の融合されたループは一度だけ走査し、中間結果はレジスタに保持し、ベクトライザには エイリアシングのない、cold error path を伴うきれいな counted loop を渡します。「素直に書いたコード」が*そのまま*速いコードなのです — 明快さのために速度を犠牲にしているわけではない、それこそがこの仕組みの要点です。

## 実例

構造体の配列に対して、在庫のある(in-stock)商品の税込価格を合計する例です。

```align
Item { price: f64, active: bool }

fn with_tax(p: f64) -> f64 = p * 1.08

fn main() -> i32 {
    items := [
        Item { price: 100.0, active: true },
        Item { price: 50.0,  active: false },
        Item { price: 200.0, active: true },
    ]
    total := items.where(.active).price.map(with_tax).sum()
    // (100.0 + 200.0) * 1.08 = 324.0
    print(total)
    return 0
}
```

ループはひとつだけ。一時変数もなし。`where(.active)` は次のイテレーションへスキップする分岐になり、`.price` はフィールドのロードになり、`with_tax` はインライン化され、`sum` はアキュムレータで加算していきます。

## 身につけるべき習慣

`for` を書こうとしていると気づいたら、立ち止まってこう問いかけてください — この処理は何を変換しているのか、と。そしてそれをパイプラインとして書いてください。明示的なループが必要なのは、パイプラインの語彙が本当に表現できない稀なケースだけです — そしてそのケースは、あなたが思うよりずっと稀です。
