# 13. The Big Crunch

> 🌐 [English](../13-the-big-crunch.md) · **日本語**

**Q1.** 10ギガバイトのログファイルを渡されました。目標は `status == 200` のログに絞って、各 `user_id` が何回出現したかを数えることです。

**A1.** 読み込み、パースし、フィルタをかけ、グループ化し、カウントする必要があります。

---

**Q2.** オブジェクト指向の直感はこう囁きます。「`file.read_to_string()` を使おう」と。

**A2.** するとマシンは悲鳴を上げます。たった今、あなたはヒープに10ギガバイトのメモリを確保し、OSのバッファからプログラムのメモリへとデータを丸写し（コピー）したからです。

---

**Q3.** では、1行ずつストリーム処理しますか？ `for line in file.lines() { ... }`

**A3.** 少しマシになりました。しかし、文字列を一つずつ `LogLine` オブジェクトにパースしていくやり方では、数千万個の小さなオブジェクトがヒープにばらまかれ、キャッシュを荒らし、パイプラインの流れを寸断してしまいます。

---

**Q4.** ならば、10ギガバイトのデータをどうやって読めと？

**A4.** 
```align
view := fs.read_bytes_view("access.log")
```
これはOSレベルのメモリマップ（`mmap`）です。ファイルがアドレス空間に直接マッピングされます。1バイトたりともコピーされません。我々はディスクを指差して、それを「メモリ」と呼んでいるだけです。

---

**Q5.** しかしそれは単なる `u8` のスライスです。どうやって列（カラム）を取り出すのですか？

**A5.** Arenaの中に、直接「横倒し（SoA）」でデコードします：
```align
arena {
    logs: soa<Log> := json.decode_many(view)?
    ...
}
```

---

**Q6.** `decode_many` は、ログ内のすべてのURLに対してヒープ上に文字列（String）をアロケーションするのですか？

**A6.** いいえ。Align の文字列は単なる `slice<u8>` です。パーサーは、すでに手元にある `view`（メモリマップされたファイル）の一部を指し示すスライスを作るだけです。コピーはゼロです。

---

**Q7.** これで `soa<Log>` である `logs` が手に入りました。必要なのは `status == 200` だけです。

**A7.** 3章で学んだことを使いましょう：
```align
valid_logs := logs.where(fn l { l.status == 200 })
```

---

**Q8.** この `valid_logs` は、データがコピーされた新しい SoA ですか？

**A8.** いいえ。これはフィルタされた「ビュー」です。メモリは一切移動していません。

---

**Q9.** 最後に、`user_id` の出現回数を数えます。

**A9.** 10章に答えがあります：
```align
results := valid_logs.user_id.group_by().count()
```

---

**Q10.** 全てを一つのパイプラインに繋げてみてください。

**A10.**
```align
view := fs.read_bytes_view("access.log")

arena {
    results := json.decode_many::<Log>(view)?
        .where(fn l { l.status == 200 })
        .user_id
        .group_by()
        .count()
        
    print(results)
}
```

---

**Q11.** このパイプラインは、いくつのヒープアロケーションを実行しましたか？

**A11.** ゼロです。OSがファイルをマッピングし、ArenaがSoAのレイアウトを保持し、コンパイラがフィルタとグループ化を「1つの高速なループ」へと融合（fuse）しました。データはメモリマップから直接カラムへと流し込まれました。

---

**Q12.** `arena` ブロックが終わるとどうなりますか？

**A12.** カラムたちは一瞬で消え去ります。ガベージコレクタが走ることも、デストラクタがツリーをたどることもありません。メモリのポインタが「リセット」されるだけです。

---

**Q13.** あなたはたった今、アロケーションを一切行わず、ループをたった1回だけ回して、10ギガバイトのデータを一瞬で処理しました。

**A13.** これが The Big Crunch（大咀嚼）です。これこそが、Align がオブジェクトも、暗黙のアロケーションも、クラスも持たない理由です。

---

> **第十三の戒律**
>
> *データをオブジェクトの形に丸めるな。データは平らに敷き詰め、大地（ディスク）から直接マッピングし、その上をパイプラインに流させよ。*
