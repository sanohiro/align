# 1. Toys

> 🌐 [English](../01-toys.md) · **日本語**

**Q1.** `42` は値ですか？

**A1.** はい。`i64` です — 文脈によって他のビット幅が要求されない限りは。

---

**Q2.** `42.0` は同じ値ですか？

**A2.** いいえ。そちらは `f64` です。Align が勝手に両者を混ぜることはありません。

---

**Q3.** `x := 42` は何をしますか？

**A3.** `x` を導入し、`42` に束縛します。`:=` は新しい変数を宣言するという意味です。

---

**Q4.** では、その直後の `x = 43` は何をしますか？

**A4.** コンパイルエラーになります。`x` が `mut` として宣言されていないからです。

---

**Q5.** `x` を変更可能にするにはどうしますか？

**A5.** `mut x := 42` とします。こうすれば `x = 43` が許可されます。可変性は宣言時に決定するものであり、後から付与するものではありません。

---

**Q6.** なぜ mutability（可変性）は誕生時に宣言しなければならないのですか？ 後から気が変わったらダメなんですか？

**A6.** 可変性とは変数の単なる「機能」ではなく、プログラマの「意図の宣言」だからです。コードのブロックを読んだとき、何がじっと動きを止め、何が激しく踊り回るのかを、あなたは一瞬で見分けられなければなりません。もし、いかなる変数も突然踊り出すことができるのだとしたら、あなたは自分が立つ床の安全性すら信じられなくなってしまうでしょう。

---

**Q7.** このブロックの値は何ですか？

```align
{
    a := 3
    a * 2
}
```

**A7.** `6`。ブロックの値は、その最後の式の値になります。ブロックの中に `return` はいりません — 最後の式がそのまま答えになります。

---

**Q8.** `if 2 > 1 { "yes" } else { "no" }` の値は何ですか？

**A8.** `"yes"`。`if` は式（expression）です。それ自体が値を持つので、変数に束縛することもできます: `ans := if 2 > 1 { "yes" } else { "no" }`。

---

**Q9.** 3を渡されたとき、この関数は何を返しますか？

```align
fn square(x: i64) -> i64 = x * x
```

**A9.** `9`。`= expr` の形は、式一つだけで関数全体を書き表す構文です。

---

**Q10.** では、こちらは3を渡されたら何を返しますか？

```align
fn cube(x: i64) -> i64 {
    return x * x * x
}
```

**A10.** `27`。ブロックの本体では `return` と書きます。本体の書き方はこの2つだけで、3つ目はありません。

---

**Q11.** `square(square(2))` は何ですか？

**A11.** `16`。関数は合成できます。知ってましたね。準備運動です。

---

**Q12.** `print(7 / 2)` は何を表示しますか？

**A12.** `3`。整数の割り算はゼロに向かって切り捨てられます。

---

**Q13.** `print(7 / 0)` は何をしますか？

**A13.** エラーを出してプログラムを止めます — 大きな音を立てて、その行で。間違った数字のまま静かに進み続けることは決してありません。

---

**Q14.** `x: i8 := 127`。では `x + 1` は何ですか？

**A14.** `-128`。整数のオーバーフローは2の補数表現でラップアラウンド（ぐるっと回って戻る）します。常に定義された、意図通りの動作です。

---

**Q15.** `[1, 2, 3]` は値ですか？

**A15.** はい — 3つの `i64` がメモリ上に「連続して」並んだ配列です。連続していること（Contiguous）が重要です。ここから先の4つの章はすべてその話になります。

---

**Q16.** `[1, 2, 3][0]` は何ですか？

**A16.** `1`。インデックスはゼロから始まり、境界チェック（bounds-checked）が行われます。

---

**Q17.** `[1, 2, 3][3]` は何ですか？

**A17.** 実行時の強制終了（abort）。範囲外アクセスはエラーであり、冒険の始まりではありません。

---

**Q18.** 最後のおもちゃです。このプログラム全体は何を表示し、どんな値で終了（exit）しますか？

```align
fn main() -> i32 {
    print("ready")
    return 0
}
```

**A18.** `ready` と表示して、`0` で終了します。`main -> i32` は C言語と同じプログラムの入り口です。戻り値がそのまま終了コード（exit code）になります。

---

> **第一の戒律**
>
> *`:=` で束縛せよ。`=` による再代入は、`mut` と宣言したものだけにせよ。*

これで準備は整いました。
