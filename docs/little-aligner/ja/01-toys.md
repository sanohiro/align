# 1. Toys

> 🌐 [English](../01-toys.md) · **日本語**

**Q1.** `42` は値ですか?

**A1.** はい。`i64` です — すぐそばの何かが別のビット幅を求めていない限り。

---

**Q2.** `42.0` は同じ値ですか?

**A2.** いいえ。そちらは `f64` です。Align があなたの知らないところで両者を混ぜることはありません。

---

**Q3.** `x := 42` は何をしますか?

**A3.** `x` を導入し、`42` に束縛します。`:=` は *新しい名前が生まれる* という意味です。

---

**Q4.** では、その直後の `x = 43` は何をしますか?

**A4.** コンパイルできません。`x` は `mut` と宣言されていないからです。

---

**Q5.** `x` は変わりうる、と言うにはどうしますか?

**A5.** `mut x := 42`。すると `x = 43` は歓迎されます。可変性は生まれたときに宣言するもの。あとから見つけるものではありません。

---

**Q6.** このブロックの値は何ですか?

```align
{
    a := 3
    a * 2
}
```

**A6.** `6`。ブロックの値は、その末尾の式です。ブロックの中に `return` は要りません — 最後の式が *そのまま* 答えです。

---

**Q7.** `if 2 > 1 { "yes" } else { "no" }` は何ですか?

**A7.** `"yes"`。`if` は式です。値を持つので、束縛できます: `ans := if 2 > 1 { "yes" } else { "no" }`。

---

**Q8.** この関数は `3` を渡すと何を返しますか?

```align
fn square(x: i64) -> i64 = x * x
```

**A8.** `9`。`= expr` の形は、関数本体まるごとを式ひとつで書いたものです。

---

**Q9.** ではこちらは、`3` を渡すと?

```align
fn cube(x: i64) -> i64 {
    return x * x * x
}
```

**A9.** `27`。ブロック本体は `return` と言います。本体の書き方は2つ。3つ目はありません。

---

**Q10.** `square(square(2))` は何ですか?

**A10.** `16`。関数は合成できます。そんなことは知っていますね。まだ準備運動です。

---

**Q11.** `print(7 / 2)` は何を表示しますか?

**A11.** `3`。整数の割り算はゼロに向かって切り捨てます。

---

**Q12.** `print(7 / 0)` は何をしますか?

**A12.** プログラムをエラーで止めます — その行で、大声で。こっそり間違った数を出したりはしません。

---

**Q13.** `x: i8 := 127`。`x + 1` は何ですか?

**A13.** `-128`。整数のオーバーフローは、二の補数でぐるりと巻き戻ります。いつでも、定義済みで、意図してそうなっています。

---

**Q14.** `[1, 2, 3]` は値ですか?

**A14.** はい — 3つの `i64` からなる配列で、メモリ上に連続して並んでいます。連続していることが肝心です。それがまるまる次の4章分のテーマです。

---

**Q15.** `[1, 2, 3][0]` は何ですか?

**A15.** `1`。添字はゼロから始まり、そして境界チェックされます。

---

**Q16.** `[1, 2, 3][3]` は何ですか?

**A16.** 実行時アボート。範囲外はエラーであって、冒険ではありません。

---

**Q17.** もう一つおもちゃを。このプログラム全体は何を表示し、何で終了しますか?

```align
fn main() -> i32 {
    print("ready")
    return 0
}
```

**A17.** `ready` を表示し、`0` で終了します。`main -> i32` は C のエントリポイント。返り値が終了コードになります。

---

> **第一の戒律**
>
> *束縛には `:=` を。再代入には `=` を、ただし `mut` なものにだけ。*

さあ、これで遊べます。
