# 5. Chains

> 🌐 [English](../05-chains.md) · **日本語**

**Q1.** これを左から右へ、声に出して読んでください:

```align
scores.map(fn s { s + 5 }).where(fn s { s >= 60 }).count()
```

**A1.** 「scores を取り、各点に5を足し、60以上のものを残し、それを数える。」読めるなら、書けます。それが chain の狙いです。

---

**Q2.** `scores := [55, 40, 90]` のとき、これは何を答えますか?

**A2.** `2`。下駄が `55` を `60` に(合格)、`40` を `45` に(不合格)持ち上げます。`90` は最初から疑いようがありません。

---

**Q3.** ステージの順序は問題になりますか? 比べてください:

```align
xs.where(fn x { x > 0 }).map(fn x { x * 2 }).sum()
xs.map(fn x { x * 2 }).where(fn x { x > 0 }).sum()
```

**A3.** 2倍にする分には、答えは一致します — 2倍しても符号は変わりませんから。でも *仕事量* は違います。前者は早めに絞り、2倍にする数が少なくて済む。できるときは早めに絞りましょう。chain は、あなたが書いたことを、書いた順にきっかり実行します。

---

**Q4.** ではこの2つは?

```align
xs.map(fn x { x - 10 }).where(fn x { x > 0 }).count()
xs.where(fn x { x > 0 }).map(fn x { x - 10 }).count()
```

**A4.** 答えが違います! `xs := [5, 15]` のとき、前者は引いてから正の数を残す — count は `1`。後者は正の数を残してから引く — count は `2`(そのあとの `where` はもう何も絞りません)。chain は文であり、語順は意味です。

---

**Q5.** 5ステージの chain はいくつのループにコンパイルされますか?

**A5.** 1つ。いつでも1つ。`map`–`where`–`map`–`scan`–`sum` — カウント付きループ1回、中間値はレジスタに、ベクトル化器の出番も整っています。確かめてもよいです: `alignc emit-llvm yourfile.align`。

---

**Q6.** では、長い chain を複数行に分けて書いたら、何かコストがかかりますか?

```align
total := items
    .where(.active)
    .price
    .map(with_tax)
    .sum()
```

**A6.** 何も。`.` で始まる行は chain を続けます。レイアウトは読み手のためのもの。コンパイラはどちらでも1つの pipeline を見ます。

---

**Q7.** chain を途中で止めて、やりかけの仕事を持っておいてもいいですか?

```align
halfway := items.where(.active).price
```

**A7.** いいえ — chain は畳み込み(`sum`、`count`、…)か実体化(`to_array`、`sort`、`map_into`)で終わらなければなりません。途中を持っておくのは、秘密の未完成ループを抱えることです。終わらせるか、始めないか。

---

**Q8.** では、絞り込んだ集合を2つの問いで *再利用* するにはどうしますか?

**A8.** 一度実体化して、二度問う:

```align
active := items.where(.active).price.to_array()
print(active.sum())
print(active.max())
```

目に見える確保が1回、安上がりな畳み込みが2回。(いつか *グループ* ごとに *たくさんの* 集計が欲しくなります — それが10章の `agg` です。)

---

**Q9.** ここで `chunks` は何をしますか?

```align
[1, 2, 3, 4, 5].chunks(2).map(fn c { c.sum() }).to_array()
```

**A9.** `[3, 7, 5]` — `[1,2]`、`[3,4]`、`[5]` の和。`chunks(n)` は配列を `n` 枚ずつの手札に配ります(最後の手札は端数)。それぞれが slice で、それぞれがほかと同じように pipeline されます。

---

**Q10.** *あなた* が所有するメモリに書き込む chain — `map_into` とは何ですか?

```align
src.map(dbl).map_into(dst)
```

**A10.** 確保ゼロの終端です。結果は slice `dst` に入ります。`dst` は同じ長さでなければならず、`src` と重ならないことをコンパイラが証明します。バッファを再利用するホットパス向けです。

---

**Q11.** さて、習慣です。あなたはこれからループを書こうとしています。まず何を問いますか?

**A11.** *「変換は何か?」* そうしたら、それをステージとして書きます: 各要素を変える → いくつか残す → 畳み込む。ステージに名前をつけられないなら、11章(再帰)が待っています — でも、まずはその問いを、毎回。

---

> **第五の戒律**
>
> *chain は左から右へ読み、早めに絞り、そして終わる。ひとつの文、ひとつのループ、ひとつの答え。*
