# 10. Count Me by Name

> 🌐 [English](../10-count-me-by-name.md) · **日本語**

**Q1.** 売上、1行に1件:

```text
east 3
west 4
east 5
```

*地域ごと* の合計は?

**A1.** east `8`、west `4`。グループ分けして、それから足しました。誰にでもできます — 問題は、ハッシュマップを手で書かずにそれを言うことです。

---

**Q2.** Align で言ってください。`k`(地域 id)と `v` を持つ soa に対して:

**A2.**

```align
g := s.group_by(.k).sum(.v)
```

一つのフィールドでグループ分けし、別のフィールドを畳み込む。この1行が分析の主力馬です。

---

**Q3.** 返ってくるのは何ですか?

**A3.** 2つの列が、ペアで: `g.0` は相異なるキー、`g.1` は各キーの合計 — 一方の行 `i` が他方の行 `i` に対応します。列が入り、列が出る。9章はずっと当てはまり続けています。

---

**Q4.** 上の売上で、`g.0.count()` と `g.1.sum()` は何ですか?

**A4.** `2`(east、west)と `12`(全売上 — グループ分けは同じ数を並べ替えただけ)。

---

**Q5.** `sum` のほかに、`group_by` の後には何が続けられますか?

**A5.** `min(.f)`、`max(.f)`、`count()` — 地域ごとの最大の売上、地域ごとの売上件数。(`count` はフィールドを取りません。数えるのに要らないからです。)

---

**Q6.** `group_by` は単独で立てますか — いまグループ分けして、あとで集計?

**A6.** いいえ — 5章と同じ掟です。裸の `group_by` は未完成の文(隠れた断片のテーブル)です。何を畳み込むか、ひと息のうちに言ってください。

---

**Q7.** キーが *名前* で、一度に3つの問い — sum、max、count。3回のパス?

**A7.** 1回:

```align
g := xs.group_by(.name).agg(sum(.a), max(.b), count())
```

`agg` はキーごとに3つすべてを1回のパスで畳み込みます: `g.0` は名前、`g.1` は sum、続いて max、count。メモリを一巡り、すべてのアキュムレータが相乗りして。

---

**Q8.** なぜ1回のパスがそれほど大事なのですか?

**A8.** 100万行ともなれば、メモリを一巡りすること *こそ* がコストだからです。3回のパスはテーブルを3度読みます。これは5章の融合と同じ教えを、分析のスケールで語ったものです。

---

**Q9.** `name` — string — でグループ分けを5回、別々にします。何が黙って高くつきますか?

**A9.** 同じ string をハッシュし比較すること、それを5回分。

---

**Q10.** 治療法は?

**A10.** 一度だけ払う:

```align
e := xs.dict_encode(.name)          // intern the names → small ids
s := e.group_by(.name).sum(.score)  // these ride the ids —
c := e.group_by(.name).count()      //   no re-hashing
```

辞書エンコーディング — 列指向データベースの最も古い手品を、目に見える呼び出し1つで。

---

**Q11.** 列指向データベースが、私たちの答えに何度も現れます。偶然ですか?

**A11.** まったく偶然ではありません。横倒しレイアウト(9)、グループ畳み込み(10)、辞書エンコーディング — 分析エンジンがこれらに収束したのは、ハードウェアがそう言い張るからです。Align の一手は、これらを *言語* にすること。だから普通のコードが、エンジンたちの行き着いた場所に着地するのです。

---

**Q12.** ドリル。`{"name":..., "a":..., "b":...}` の行からなる JSON 文字列から、相異なる名前と、名前ごとの最大の `b` を — 1回のパスで。

**A12.**

```align
xs: array<Row> := json.decode(data)?
g := xs.group_by(.name).agg(max(.b), count())
print(g.0.count())      // how many names
print(g.1.max())        // the largest of the per-name maxima
```

(decode、group、fold — テキストから答えまで3行。`count()` はタダで相乗りしました。)

---

> **第十の戒律**
>
> *グループ分けと畳み込みは、ひと息で。問いはすべて1回のパスで尋ね、string のキーには一度だけ代価を払え。*
