# 6. One of Many

> 🌐 [English](../06-one-of-many.md) · **日本語**

**Q1.** 信号は赤か黄か緑です。同時に2つになることはありますか?

**A1.** いいえ。多くのうちの一つ — それが sum type です:

```align
Light { Red, Yellow, Green }
```

---

**Q2.** 緑のものはどう作りますか?

**A2.** `Light.Green`。いつでもその型で修飾して — 裸の `Green` は誰のものにもなりえますから。

---

**Q3.** いまどれを持っているかは、どう尋ねますか?

**A3.** 尋ねません — `match` します:

```align
fn go(l: Light) -> i64 = match l {
    Red    => 0,
    Yellow => 0,
    Green  => 1,
}
```

---

**Q4.** アームの中で、なぜ `Light.Red` ではなく `Red` なのですか?

**A4.** `match` の中では、対象(scrutinee)の型が分かっています。アームは裸のバリアント名で語ります。外では、構築は修飾したままです。

---

**Q5.** `Green` のアームを消したら、どうなりますか?

**A5.** コンパイルエラー。match がすべてのバリアントを覆わなくなったからです。**網羅性が契約です。** 来年バリアントを増やせば、プログラム中のすべての `match` が手を挙げます。

---

**Q6.** `Red` と `Yellow` は答えを共有します。アームを2つ書かねばなりませんか?

**A6.** 1つのアームが両方を持てます: `Red | Yellow => 0`。あるいは残りを掃く: `_ => 0` — でも掃くなら承知の上で。`_` は、まだ発明されていないバリアントまで飲み込みます。

---

**Q7.** バリアントは何かを運べますか?

```align
Shape { Circle(i64), Rect(i64, i64), Dot }
```

**A7.** はい — ペイロードを。`Shape.Circle(10)` は *半径を中に入れた* 円です。`Shape.Dot` は何も運びません。

---

**Q8.** ペイロードはどうやって取り出しますか?

**A8.** match のアームがそれに名前をつけます:

```align
fn area(s: Shape) -> i64 = match s {
    Circle(r)  => 3 * r * r,
    Rect(w, h) => w * h,
    Dot        => 0,
}
```

`area(Shape.Rect(3, 4))` は `12`。ペイロードは、パターンがそれを捕まえた場所にだけ存在します — Rect かもしれないものに `.radius` を呼ぶことはできません。

---

**Q9.** `match` は文ですか、式ですか?

**A9.** 式です。Q8 の関数本体まるごとが式そのものです。束縛し、返し、渡せます: `verdict := match s { ... }`。

---

**Q10.** 数に対して match してもよいですか? `match n { 0 => ..., _ => ... }`

**A10.** いいえ。`match` は多くのうちの一つの型のためのもの。数には `if` を使います。道具は2つ、それぞれで完結。中途半端に重なって、覚えることを増やしたりはしません。

---

**Q11.** ペイロードには何を入れられますか?

**A11.** スカラーと素朴な構造体 — `Wrap(Point)` は大丈夫で、アーム `Wrap(p) => p.x + p.y` は中に手を届かせます。(`string` のような所有型のペイロードは、今は受け付けません。)

---

**Q12.** これをモデル化してください: 取得中のページは *読み込み中* か、*サイズ付きで準備完了* か、*コード付きで失敗* か。

**A12.**

```align
Page { Loading, Ready(i64), Failed(i64) }

fn describe(p: Page) -> i64 = match p {
    Loading   => 0,
    Ready(n)  => n,
    Failed(c) => -c,
}
```

ありえない状態 — 準備完了 *かつ* 失敗、ページのないサイズ — は書けません。それこそが sum type の *目的* です。

---

> **第六の戒律**
>
> *あるものが多くのうちの一つなら、その多くを言え。それから `match` し、リストの管理はコンパイラに任せよ。*
