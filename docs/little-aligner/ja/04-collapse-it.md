# 4. Collapse It

> 🌐 [English](../04-collapse-it.md) · **日本語**

**Q1.** `sum` と `count` にはもう出会いました。`[5, 3, 8, 1].min()` は何ですか？

**A1.** `1`。

---

**Q2.** そして `[5, 3, 8, 1].max()` は？

**A2.** `8`。ここまでの reducer は `sum`、`count`、`min`、`max`。どれも複数の値を一つにまとめます。

---

**Q3.** `[1, 2, 3, 4].any(fn x { x > 3 })` は何ですか？

**A3.** `true` — 少なくとも一つの要素が通ります。兄弟分の `all(fn x { x > 0 })` も、ここでは `true`。`any` と `all` は `bool` に畳み込まれます。

---

**Q4.** `[2, 4, 6].all(fn x { x % 2 == 0 })` は何ですか？

**A4.** `true`。では `[2, 4, 7]` は? — `false`。`7` が条件を満たさないからです。そして `all` は、それを見つけた瞬間に結果を出せるのです。

---

**Q5.** ぴったりの reducer がないとしましょう。今度は *積* が欲しい。どうしますか？

**A5.** 万能なツール: `reduce`。

```align
[1, 2, 3, 4].reduce(1, fn acc, x { acc * x })
```

`24`。

---

**Q6.** なぜ先頭に `1` があるのですか？

**A6.** 初期値です — まず種（seed）、それから畳み込む関数。`acc` は `1` から始まり、各要素が掛け合わされます。和ならば `0` を種にするでしょう。種は、空の配列が返す答えでもあります。

---

**Q7.** `fn acc, x { acc * x }` で、どちらがどちらですか？

**A7.** `acc` はこれまでの答え、`x` は次の要素。lambda は、次の要素をどう答えに組み込むかを指定しています。

---

**Q8.** `reduce` を使って、`sum` を自分で書いてください。

**A8.** `xs.reduce(0, fn acc, x { acc + x })`。これで分かりました — `sum`、`count`、`min`、`max` はただのショートカットで、`reduce` こそが万能のエンジンなのです。名前付きのものは意図を素早く伝えるためのものです。

---

**Q9.** `[1, 2, 3, 4].scan(0, fn acc, x { acc + x })` は何ですか？

**A9.** *途中経過* の和: `1, 3, 6, 10`。`scan` は途中の計算過程を見せる `reduce` です — だからこれはあくまで途中段階であり、最終結果ではありません。まだ何かがこれを消費しなければなりません。

---

**Q10.** では `[1, 2, 3, 4].scan(0, fn acc, x { acc + x }).max()` は何ですか？

**A10.** `10` — 途中経過の和が到達した最大値。(配列に負の数があると、この問いは面白くなります。いわば最高到達点です。)

---

**Q11.** `[3, 1, 2].map(fn x { x }).sort()` は何ですか — 順序を頼んでもいいですか？

**A11.** `[1, 2, 3]`、実体化された配列です。`sort` は並べ替えるために配列を作らねばなりません — だから `to_array` と同じく、本物の配列を手渡します。メモリ確保は隠さずに行われます。

---

**Q12.** そして `[10, 21, 32, 3].sort_by_key(fn x { -x })` の結果は？

**A12.** `[32, 21, 10, 3]`。符号を反転した値で並べ替える: `reverse` を一切使わずに降順です。

---

**Q13.** `[1, 2, 3, 4, 5].partition(fn x { x % 2 == 0 })` は何ですか？

**A13.** 一度に2つの配列: `([2, 4], [1, 3, 5])` — 条件を満たしたものと、そうでないもの。まとめて受け取りましょう: `(evens, odds) := ...`。

---

**Q14.** 空の slice に対して、`any` と `all` は何を返しますか？

**A14.** `any` は `false`、`all` は `true`。条件を満たす証人はおらず、条件を破る反例もいません。

---

**Q15.** 何もないものの `all` が `true` なのは、なぜ？

**A15.** 「規則を破る要素を見つけたか？」と問い直してください。空なら見つけていません。この identity により、`sum` が `0`、積が `1` から始まるのと同じように、特別な空 case なしで合成できます。

---

**Q16.** `[1, 2, 3, 4]` の2乗和は？

**A16.**

```align
[1, 2, 3, 4].map(fn x { x * x }).sum()
```

`30`。まず各要素が何になるかを言い、次に多数を一つへする方法を言います。

---

**Q17.** 偶数の square を数えます。先に square にする必要は？

**A17.** ありません。数とその square の偶奇は同じです。

```align
[1, 2, 3, 4].where(fn x { x % 2 == 0 }).count()
```

`2`。最後の問いが必要としない値を計算してはいけません。

---

**Q18.** sum と count を一度の fold で求めてください。

**A18.**

```align
pair := [2, 4, 6].reduce((0, 0), fn acc, x {
    (acc.0 + x, acc.1 + 1)
})
```

`pair` は `(12, 3)`。accumulator は tuple にできます。途中までの答えは、複数の pocket を持てます。

---

**Q19.** では整数の平均は？

**A19.** `pair.0 / pair.1`、つまり `4`。reduction が事実を集め、最後の式が解釈します。入力が空かもしれないなら、割る前に空の場合を決めます。

---

**Q20.** `[3, -5, 4, 2]` の途中の合計は？

**A20.** `[3, -2, 2, 4]`。

```align
[3, -5, 4, 2].scan(0, fn acc, x { acc + x }).to_array()
```

---

**Q21.** 途中の合計の最大値は？

**A21.**

```align
[3, -5, 4, 2].scan(0, fn acc, x { acc + x }).max()
```

`4`。stage が途中の答えを見せ、後ろの reducer がその履歴について問えます。

---

> **第四の戒律**
>
> *まず種、それから畳め。そして名前つきの reducer のほうが意図が明確になるなら — `sum`、`count`、`min`、`max`、`any`、`all` — はっきり言え。*
