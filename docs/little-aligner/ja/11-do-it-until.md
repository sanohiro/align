# 11. Do It Until

> 🌐 [English](../11-do-it-until.md) · **日本語**

> *この章の `loop` 式は設計も仕様も済んでいますが、コンパイラの実装はまだ
> 追いついていません — **implementation in progress**。あと数週間は、自分がコンパイラに
> なったつもりで読んでください。*

**Q1.** Align の `for` はどこにありますか?

**A1.** ありません。`while` もありません。あなたは 10 章のあいだ、それに気づかずやってきました — パイプラインこそがループだったのです。

---

**Q2.** でも、本当に一歩ずつ進む処理もあります。ファイルを 4096 バイトずつ、空になるまで読むには?

**A2.** この言語が持つ、たったひとつのループです。

```align
fn pump(r: reader, w: writer, buf: buffer) -> Result<(), Error> {
    loop {
        n := r.read(buf)?
        if n == 0 { break Ok(()) }
        w.write(buf.bytes())?
    }
}
```

`loop` は、`break` が走るまでブロックを繰り返します。空になるまで — そして外へ。

---

**Q3.** そこの `break Ok(())` は何をしましたか?

**A3.** 一度に 2 つのことを。ループを終わらせ、そして値を手渡しました。`loop` は**式**です — `if` のように、`match` のように。ループは関数の最後の式なので、ループの値がそのまま関数の答えになります。

---

**Q4.** では、reader が渡してくるものをすべて合計するには?

**A4.**

```align
mut total := 0
sum := loop {
    n := r.read(buf)?
    if n == 0 { break total }
    total = total + n
}
```

途中までの答えは、ループの*前*に宣言した `mut` のローカル変数に宿ります。`break total` が、周回を終えたときにそれを外へ運び出します。

---

**Q5.** Q2 のように、`?` と `loop` はひとつの関数で同居できますか?

**A5.** たったいま同居していました。`?` は**関数**から抜けます — どんな失敗も、いっぺんに `Err` となって全体から出ていきます。`break` は**ループ**から抜けます。2 つの扉、はっきりと札が掛かっていて、決して同じ扉ではありません。

---

**Q6.** `continue` はどこですか?

**A6.** ありません。次の周回へスキップしたいなら、本体の残りを `if` で囲みます。2 つのループを一度に抜けたいなら — その内側のループは、関数になりたがっています。出口はひとつ、`break` です。

---

**Q7.** あなたは `loop` の中で `i` を `0` から `len` まで数え、毎周 `xs[i]` に同じことをしています。何をしてしまいましたか?

**A7.** 変な帽子をかぶった `for` ループを書いてしまいました。脱がせましょう — それは最初から `xs.map(...)` だったのです。パイプラインは*データ*を受け持ち、`loop` は*制御*を受け持ちます。コンパイラもそう言います(lint が付きます)。

---

**Q8.** では再帰は? 昔の本は、自分自身を呼ぶ関数こそがループだと言っています。

**A8.** ここでは違います。再帰による「ループ」は、周回ごとにスタックフレームを 1 つ使います — Align は末尾呼び出しの魔法を約束しませんし、そもそもスコープ末尾の drop と `?` が、そうした魔法をこっそり壊してしまいます。千回のひと口が、千個のフレームを費やしてはなりません。`loop` はそのためにあります。

---

**Q9.** では、再帰は何のためにありますか?

**A9.** 再帰の*かたち*をした問題 — 入れ子になるもの — のためです。パーサの中の括弧の中のパーサ。枝がまた木を抱える木構造。**データ**が再帰するとき、関数もそうしてよいのです。再帰はかたちのためであって、回数のためではありません。

---

**Q10.** いつパイプラインで、いつ `loop` で、いつ再帰するのか。最後の組分け帽子です。

**A10.**

- 多くの要素に同じ操作 → **パイプライン**(2〜5 章)。
- グループごとの畳み込み → **`group_by`**(10 章)。
- 何周するか、始めてみるまでわからない — 空になるまでひと口ずつ、うまくいくまでリトライ、収束させる → **`loop`**、`break value` を手に。
- データそのものが入れ子になる → **再帰**、そのかたちのために。

---

> **第十一の戒律**
>
> *多くには pipeline を、until には `loop` を、入れ子には再帰を。そして、そもそもループする前に — それがパイプラインではなかったか、と問え。*

---

**Q11.** これで終わりですか?

**A11.** ドリルは、はい。Align は — [ガイド](../../guide/ja/README.md)に残りがあります: 文字列、JSON、並列処理、SIMD、標準ライブラリ。でもあなたはもう、Align を外国語のようには読みません。メニューのように読むのです。

*さあ、何か作りに行きましょう。*
