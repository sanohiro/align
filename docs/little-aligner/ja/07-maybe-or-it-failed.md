# 7. Maybe, or It Failed

> 🌐 [English](../07-maybe-or-it-failed.md) · **日本語**

**Q1.** `[1, 3, 5]` の中の偶数のリスト — その最初の要素は何ですか?

**A1.** ありません。そして「ない」ということにも型が要ります: `Option<i64>` — `Some(n)` か `None` のどちらか。

---

**Q2.** `None` は `null` と同じですか?

**A2.** いいえ。そしてその違いこそがすべてです。`null` はどんな参照の内にも潜み、待ち構えます。`None` は `Option` の内にだけ住み、そして型システムは、それがないときに何が起きるかをあなたが言うまで、`i64` を手渡しません。

---

**Q3.** では `Some(5)` から数を取り出すにはどうしますか?

**A3.** 扉は2つ。手早いほう: `x := maybe else 0` — ペイロード、さもなくばあなたの代替値。念入りなほう: `match` で、`Some(n) =>` と `None =>`、いつものように網羅的に。

---

**Q4.** `safe_head` が `Option<i64>` を返すとして、`safe_head([1, 2, 3]) else -1` は何ですか?

**A4.** `1`。そして空の slice に対しては `-1`。「ない」ことの意味は呼び手が選びました — 関数が推測する必要はなかったのです。

---

**Q5.** さて失敗です。ファイルを読もうとする関数は、どんな型を返しますか?

**A5.** `Result<string, Error>` — `Ok(contents)` か `Err(why)` のどちらか。失敗は *値* であり、理由がその中に入っています。

---

**Q6.** `Option` と `Result` の違いは何ですか?

**A6.** `None` は普通の答えです(「最初の偶数はない」 — 結構)。`Err` は事情のある失敗です(`NotFound`、`Denied`…)。「ない」ことがありふれているなら `Option`。誰かが *なぜ* を知る必要があるなら `Result`。

---

**Q7.** ここに、失敗しうる別の関数を呼ぶ、失敗しうる関数があります:

```align
fn load(path: str) -> Result<i64, Error> {
    data := fs.read_file(path)?
    return Ok(data.len())
}
```

`?` は何をしていますか?

**A7.** エラーモデルまるごとを、たった1文字で。`read_file` が `Ok(s)` で返ってきたら `data` は `s` で、続行します。`Err(e)` なら、いますぐ `Err(e)` を呼び手へ **return** します。取り出すか、抜けるか。

---

**Q8.** では、最終的に誰がエラーを *処理* するのですか?

**A8.** できる者が。各層は、それを上へ渡す(`?`)か、`match` と腰を据えて向き合うか、どちらかです。いちばん上、`main() -> Result<(), Error>` で、逃げ切った `Err` が非ゼロの終了コードに変わります。エラーはずっと値として旅をします — 目に見えない巻き戻しも、遠隔地での catch もなしに。

---

**Q9.** どうでもいい `Result` は、ただ無視してもいいですか?

```align
fs.write_file("log.txt", "hi")
```

**A9.** コンパイラはノーと言います — *未処理の Result*、これはハードエラーです。処理してください(`?`、`match`、あるいは束縛して決める)。Align では失敗してもよいのですが、*黙って* 失敗することは許されません。

---

**Q10.** 組み込みの `Error` には何が入っていますか?

**A10.** OS が語るカテゴリ — `NotFound`、`Invalid`、`Denied` — と、残りのための `Code(n)`。ほかの sum type と同じように `match` します(6章で学んだとおり。`Error` はただの、記章つきの sum type です)。

---

**Q11.** では、自分のエラー型 — そして継ぎ目をまたぐ `?` は?

**A11.** 一つ宣言して(`ParseErr { Empty, BadChar }`)、境界で **目に見えて** 変換します:

```align
v := inner(n).map_err(to_error)?
```

`?` はひとりでに型を変換したりしません。`map_err` が、`ParseErr` が `Error` になった場所を、読み手にきっちり示します。

---

**Q12.** `Result` に `else` を — 使えますか?

**A12.** いいえ。`else` は Option のものです。`Result` は理由を運び、理由は読むためのもの — `match` するか、`?` で先へ渡してください。(それでも代替値が欲しい? なら `match` で正直に2行、そう書いてください。)

---

> **第七の戒律**
>
> *「ない」は `Option`、失敗は `Result`。失敗は `?` で上へ渡し、`Result` を一つも床に落とすな。*
