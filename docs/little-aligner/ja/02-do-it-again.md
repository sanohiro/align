# 2. Do It Again

> 🌐 [English](../02-do-it-again.md) · **日本語**

**Q1.** ここに配列があります: `[1, 2, 3]`。各要素を2倍にして言ってみてください。

**A1.** `2, 4, 6`。いま頭の中で `map` をしましたね。

---

**Q2.** では Align で。`[1, 2, 3].map(fn x { x * 2 }).sum()` は何ですか?

**A2.** `12`。`map` が各要素を2倍にし — `[2, 4, 6]` — `sum` がそれらを足しました。

---

**Q3.** `fn x { x * 2 }` は何ですか?

**A3.** lambda です。名前のない関数。`fn`、引数、ブロック。引数の型は、流れ込んでくる要素から推論されます。

---

**Q4.** 名前つきの関数を代わりに使えましたか?

**A4.** はい:

```align
fn double(x: i64) -> i64 = x * 2

[1, 2, 3].map(double).sum()
```

答えは同じ `12`。ステージは、形さえ合えばどんな関数でも受け取ります。

---

**Q5.** `[1, 2, 3].map(fn x { x * x }).sum()` は何ですか?

**A5.** `14`。2乗して: `1 + 4 + 9`。

---

**Q6.** `[].map(fn x { x * x }).sum()` — 何もないものを map したら?

**A6.** ひっかけ問題です — 空の配列リテラルには、推論すべき要素型がありません。でも `i64` の空の *slice* を map すれば、和は `0`。無を2倍しても、やはり無です。

---

**Q7.** lambda は、自分の外にある名前を使えますか?

```align
factor := 3
[1, 2, 3].map(fn x { x * factor }).sum()
```

**A7.** はい — `18`。lambda は `factor` を値で**キャプチャ**します。作られた瞬間にコピーを取るのです。

---

**Q8.** もし `factor` が `mut` で、`map` の *あと* に変わったら、map はそれを見ますか?

**A8.** いいえ。**値で**キャプチャ — コピーですから。変化させられる共有環境などありません。10章の並列の世界でこれを思い出してください。何も競合しないのはこれが理由です。

---

**Q9.** `[1, 2, 3].map(fn x { x * 2 })` — `.sum()` なしでは何ですか?

**A9.** コンパイルできません。pipeline は終わらなければなりません — 畳み込みで、あるいは実体化で。宙ぶらりんの半端な pipeline は隠れたコストであり、Align は隠しごとをしません。

---

**Q10.** では、2倍にした配列を本当に *取っておきたい* なら、どうしますか?

**A10.** そう言ってください: `[1, 2, 3].map(fn x { x * 2 }).to_array()`。これで `[2, 4, 6]` が手に入り、確保(アロケーション)は誰の目にも見える形で書かれます。

---

**Q11.** `[1, 2, 3].map(fn x { x * 2 }).map(fn x { x + 1 }).sum()` は何ですか?

**A11.** `15`。map を2つ続けて: `[3, 5, 7]`。そしてここが秘密です — コンパイラは**ループを1つ**作ります。2つではありません。ステージは融合します。

---

**Q12.** Q11 は途中でいくつ配列を作りましたか?

**A12.** ゼロ。`3`、`5`、`7` はそれぞれ一瞬だけレジスタに宿り、すぐさま足し込まれました。だからこそ、途中を取っておくのではなく、pipeline を終わらせるのです。

---

**Q13.** map の中で `print` は何をしますか?

```align
[1, 2].map(fn x { print(x) ... })
```

**A13.** やめましょう。pipeline のステージは *計算する* ためのもの。そしてあとで — 10章で — 純粋性をきっぱり要求するステージに出会います。表示は、pipeline が答えを手渡してくれたあとで。

---

> **第二の戒律**
>
> *各要素を訪ねたいなら、`map` せよ。結果を取っておきたいなら、`to_array()` と声に出して言え。*
